3. WHAT'S NEW IN 9.3
New in 9.3 Highlights for developers and administrators

Database Agent Service Revamp

The DBQueue Processor is now a fully .NET-based component. The old SQL stored procedure QBM_PWorkDBQueueMain is no longer required or called. This improves throughput and eliminates SQL Agent dependency for process dispatch.

CEF Audit Logging

OIM now automatically generates Common Event Format (CEF) log messages for defined object changes and sends them to a syslog server.

ObjectDescription
QBMCEFDefinitionsDefines change types, message templates, and up to 5 replacement parameters in ObjectWalker notation
Config keyQBM | CEF | SyslogServer, QBM | CEF | SyslogPort

Microsoft 365 Email via Entra ID

Email notifications now support Microsoft 365 OAuth delivery. Configure via Common | MailNotification | O365ClientId after registering an app in Microsoft Entra ID. No SMTP relay required.

Process Archive View in Job Queue Info

The Job Queue Info tool now shows a Process Archive tab alongside the live queue, displaying completed (including archived) processes when a History Database is configured. Filterable by status, date range, and component.

Attestation Enhancements

  • Terms of use can now be attached as a PDF file per attestation policy
  • Multi-language support for terms of use text
  • Random sample generation for sample-based attestation policies

UI & Permissions Editor

  • Multiple result lists per menu item with easy switching
  • Customisable column display and reordering in result lists
  • New Designer task: Add via authorization object
  • Two different variations of the same authorization object allowed in one definition

5. XOBJECTKEY & SYSTEM COLUMNS
XOBJECTKEY ANATOMY

Every OIM entity has a unique XObjectKey which is used for cross-table references, process parameters, and API operations.

<Key><T>TableName</T><P>UID-value-here</P></Key> <!-- Example: a PersonInOrg row --> <Key><T>PersonInOrg</T><P>abc123-def456-...</P></Key>

In C# API code, test the table with objectKey.StartsWith("<Key><T>PersonInOrg</T>", StringComparison.OrdinalIgnoreCase).

XORIGIN BITMASK

XOrigin on assignment tables (PersonInOrg, PersonInAERole, ADSAccountInADSGroup, etc.) tracks how an assignment was made. It is a bitfield — multiple bits can be set simultaneously.

BitHexMeaningHow to remove
10x01Direct assignment (manually assigned in Designer / API)MarkForDeletion() + commit
20x02Inherited from parent role in hierarchy (community-confirmed)Remove membership from parent
40x04Dynamically assigned (rule-based) (community-confirmed)Change rule or identity attributes
80x08Requested via IT Shop (PersonWantsOrg) (community-confirmed)Call Unsubscribe on PersonWantsOrg
160x10FromGroup — assigned via group/federation membership (source: AAD_FGIBitPatternXOrigin SQL function)Remove from the source group/federation
Important Never delete an XOrigin=8 assignment directly. Always unsubscribe through PersonWantsOrg. Direct deletion leaves the IT Shop request in an inconsistent state.

6. TEMPLATES (TPL)

Templates automatically provision target-system accounts (e.g., Active Directory) based on identity data. They define what to create and when.

Key Concepts

ConceptDescription
Account DefinitionLinks an OIM identity to a target-system account type. Controls who gets an account and under which conditions.
TemplateDefines default values for account properties (SamAccountName, OU, etc.) when creating or updating accounts.
Automatic Creation ModeControls if accounts are created automatically on assignment, manually, or for certain identity types only.
Mapping RuleA VB.NET inline expression transforming an identity property into a target-system property at creation time.
TemplateOverwriteControls whether the template re-applies on subsequent updates (overwrite) or only runs at initial creation.

VB.NET Expression Examples

' Build SamAccountName: FirstInitial + LastName, lowercase Left($FirstName:String$, 1).ToLower() & $LastName:String$.ToLower() ' Display name: "LastName, FirstName" $LastName:String$ & ", " & $FirstName:String$ ' OU path based on department "OU=" & $UID_Department_Display:String$ & ",OU=Users,DC=company,DC=com" ' Conditional OU: external vs internal IIf($IsExternal:Bool$, "OU=External,DC=company,DC=com", "OU=Internal,DC=company,DC=com") ' Account move on department change "OU=" & $Department_DepartmentName:String$ & ",OU=Users,DC=company,DC=com"
Tip Template inheritance: Child account definitions inherit template rules from their parent. To override a specific rule in the child, add an explicit rule with the same property name — the child rule takes precedence.

7. PROCESSES (PRO)

Processes are event-driven automation chains. When an object changes, the OIM event system enqueues a process chain in JobQueue for a Job Server to pick up.

ElementDescription
ProcessNamed chain of process steps triggered by an object event
Process StepIndividual action: run a script, call external program, send email, execute SQL
EventTrigger: Insert, Update, Delete, or custom named events
Process ParameterData passed from the triggering object to steps (e.g., $ObjectKeyBase, $UID_Person$)
JobQueuePending processes; Ready2EXE = 'TRUE' means available for pickup
Scheduled TriggerProcess chains can also be triggered on a cron-style schedule via the Scheduler object in Designer

Common VB.NET Script Step Patterns

' Fire a custom event from another process step Dim obj As IEntity = Session.Source.GetSingle("Person", "UID_Person = '" & uid & "'") obj.FireEvent("CCC_MyCustomEvent") Session.Source.Save(obj) ' Read process parameters inside a script step Dim objectKey As String = Value("$ObjectKeyBase$") Dim uidPerson As String = Value("$UID_Person$") ' Conditional logic — abort process step If condition Then RaiseError("Condition not met — aborting step") End If
Note Enqueue from C# API code: Use u.GenerateAsync("ProcessName", new Dictionary<string,string> {{ "UID_Person", uid }}, ct) inside a UnitOfWork to programmatically enqueue a process chain.

DBQueue Processing (9.3)

In 9.3 the DBQueue Processor is a .NET service component — no SQL Agent job required. Monitor queue health in Job Queue Info. If provisioning is stalled, check: (1) DBQueue service running, (2) JobQueue.Ready2EXE = 'TRUE' rows exist, (3) Job Server service is alive, (4) server functions are correctly assigned.

PROVISIONING CHAIN — STEP BY STEP
StepWhat Happens
1. Change committedA column value changes on a provisioning-tracked table (e.g., ADSAccount.IsLocked). OIM writes a row to JobQueue.
2. DBQueue Processor.NET component scans JobQueue for Ready2EXE='TRUE' entries and dispatches them to the appropriate Job Server queue.
3. Job Server picks upThe Job Server service polls its queue. It matches the process step's server function (e.g., ADSConnector) to decide which server handles it.
4. Connector executesThe connector (AD Connector, LDAP connector, etc.) translates the OIM change into a target system operation (LDAP modify, Graph API call, etc.).
5. ConfirmationOn success the process step is marked complete; on failure it enters an error state and may retry.
Important Troubleshooting stalled provisioning: (1) Is the DBQueue service running? (2) Does the JobQueue row have Ready2EXE='TRUE'? (3) Is the Job Server service alive and polling? (4) Does the Job Server have the correct server function assigned? (5) Is the target system reachable? (6) Are the Person and account both active (not IsInActive=1)?

11. ROLE MODEL

Three Role Types

TypeTablePurpose
Business RolesOrgMap company structures (projects, departments, groups). Members inherit all resources assigned to the role. Hierarchical — child roles inherit from parents.
Application RolesAERoleControl who can administer OIM itself — who can create business roles, who are attestors, who are role approvers. Built-in groups like vi_4_AUDITING_AUDITOR.
System RolesESetPackaged collections of company resources frequently assigned together. Assigned via IT Shop or directly. CAN be hierarchical — child system roles inherit from parents via ESetCollection.

Business Role Inheritance Engine

When a person is a member of a Business Role (PersonInOrg), OIM's inheritance engine walks the role hierarchy and grants all resources assigned to every ancestor role. The view PersonInBasetree (and its variants) materialises this inheritance. The resulting assignments get XOrigin = 2 (inherited).

Note Key distinction: PersonInOrg is the direct/requested membership table. PersonHasRole views represent the effective (inherited + direct) memberships. When writing queries to check if a person has effective access to a role, always query the view, not the base table.

18. API PLUGIN ARCHITECTURE

OIM 9.3 exposes data via a RESTful CompositionAPI built on ASP.NET Core. Custom functionality is added by deploying plugin DLLs that the API Server discovers at startup.

28. BULK ACTIONS (024_002)

Bulk Actions let the Web Portal process CSV files row-by-row against custom API endpoints. The portal calls four endpoints in a fixed sequence.

1 - startaction

/startaction

Pre-flight. Receives header names + total row count. Returns { message, permission, ActionAttributes }. Creates shared state (e.g., a ShoppingCartOrder). ActionAttributes is forwarded to all subsequent calls.

2 - validate

/startValidate → /validate

Called once per row. Returns object[] with { column } for valid columns or { column, errorMsg } for invalid ones. Does not write to DB. May add ValidationAttributes per row.

3 - action

/action

Called once per valid row. Performs the actual DB write. Receives all column values + ActionAttributes + per-row ValidationAttributes.

4 - endaction

/endaction

Post-flight. Receives import stats (successCount, failCount, errorRows). Returns a summary message string. Clean up shared state here.

function Get-OIMPersons { param([string]$ConnectionString, [string]$UID_Person = $null) $conn = New-Object System.Data.SqlClient.SqlConnection $ConnectionString $cmd = $conn.CreateCommand() $cmd.CommandText = @" SELECT UID_Person, FirstName, LastName, CentralAccount FROM dbo.Person WHERE (@UID IS NULL OR UID_Person = @UID) AND IsInActive = 0 "@ $cmd.Parameters.AddWithValue("@UID", [object]([string]::IsNullOrEmpty($UID_Person) ? [DBNull]::Value : $UID_Person)) | Out-Null try { $conn.Open() $reader = $cmd.ExecuteReader() $idxUID = $reader.GetOrdinal("UID_Person") $idxFN = $reader.GetOrdinal("FirstName") $idxLN = $reader.GetOrdinal("LastName") $results = [System.Collections.Generic.List[PSCustomObject]]::new() while ($reader.Read()) { $results.Add([PSCustomObject]@{ UID_Person = $reader.GetValue($idxUID) FirstName = $reader.GetValue($idxFN) LastName = $reader.GetValue($idxLN) }) } return $results } finally { if ($reader) { $reader.Close() } if ($conn) { $conn.Close() } } }
