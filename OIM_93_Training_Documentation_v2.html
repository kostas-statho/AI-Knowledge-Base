🏗️ Architecture #

OIM 9.3 is composed of several tiers communicating via a central SQL database and service bus.

ComponentRole
JobServerExecutes process chains, handles provisioning events, runs scheduled jobs. One or more instances with distinct queues and server functions.
API ServerASP.NET Core host serving the CompositionAPI. Plugin DLLs matching *.CompositionApi.Server.Plugin.dll are auto-discovered at startup.
Web PortalAngular SPA consuming the API Server. Renders UI for IT Shop requests, attestations, data explorer, and reports.
Sync EngineSynchronises external target system data (AD, LDAP, SAP, Azure AD) into OIM's internal model via Sync Editor projects.
DesignerWindows client for configuring schemas, templates, processes, scripts, workflows, and permission groups.
SQL DatabaseCentral data store. All objects, logs, queue entries, and configuration reside here. MS SQL Server.
DBQueue ProcessorSQL Server Agent job (or .NET service in 9.3) that picks queued processes from JobQueue and dispatches them to Job Server instances.

Provisioning Data Flow

  OIM Designer / Web Portal
         │  PutValueAsync / CommitAsync
         ▼
   SQL Database (central)
         │  DBQueue Processor picks up work
         ▼
   JobServer (one or more, by queue)
         │  Server Functions: ADSConnector, etc.
         ▼
   Connector (AD Connector, LDAP, …)
         │  LDAP / Graph API / etc.
         ▼
   Target System (Active Directory, Azure AD, SAP, …)
Namespace convention: All custom plugin code uses namespace QBM.CompositionApi and references DLLs from C:\Program Files\One Identity\One Identity Manager\. Projects target .NET 8.0.

🆕 What's New in 9.3 #

New in 9.3 — highlights for developers and administrators

Database Agent Service Revamp

The DBQueue Processor is now a fully .NET-based component. The old SQL stored procedure QBM_PWorkDBQueueMain is no longer required or called. This improves throughput and eliminates SQL Agent dependency for process dispatch.

CEF Audit Logging

OIM now automatically generates Common Event Format (CEF) log messages for defined object changes and sends them to a syslog server.

ObjectDescription
QBMCEFDefinitionsDefines change types, message templates, and up to 5 replacement parameters in ObjectWalker notation
Config keyQBM | CEF | SyslogServer, QBM | CEF | SyslogPort

Microsoft 365 Email via Entra ID

Email notifications now support Microsoft 365 OAuth delivery. Configure via Common | MailNotification | O365ClientId after registering an app in Microsoft Entra ID. No SMTP relay required.

Process Archive View in Job Queue Info

The Job Queue Info tool now shows a Process Archive tab alongside the live queue, displaying completed (including archived) processes when a History Database is configured. Filterable by status, date range, and component.

Attestation Enhancements

  • Terms of use can now be attached as a PDF file per attestation policy
  • Multi-language support for terms of use text
  • Random sample generation for sample-based attestation policies

UI & Permissions Editor

  • Multiple result lists per menu item with easy switching
  • Customisable column display and reordering in result lists
  • New Designer task: Add via authorization object
  • Two different variations of the same authorization object allowed in one definition

🔑 XObjectKey & System Columns #

XObjectKey Anatomy

Every OIM entity has a unique XObjectKey which is used for cross-table references, process parameters, and API operations.

<Key><T>TableName</T><P>UID-value-here</P></Key>

<!-- Example: a PersonInOrg row -->
<Key><T>PersonInOrg</T><P>abc123-def456-...</P></Key>

In C# API code, test the table with objectKey.StartsWith("<Key><T>PersonInOrg</T>", StringComparison.OrdinalIgnoreCase).

XOrigin Bitmask

XOrigin on assignment tables (PersonInOrg, PersonInAERole, ADSAccountInADSGroup, etc.) tracks how an assignment was made. It is a bitfield — multiple bits can be set simultaneously.

BitHexMeaningHow to remove
10x01Direct assignment (manually assigned in Designer / API)MarkForDeletion() + commit
20x02Inherited from parent role in hierarchyRemove membership from parent
40x04Dynamically assigned (rule-based)Change rule or identity attributes
80x08Requested via IT Shop (PersonWantsOrg)Call Unsubscribe on PersonWantsOrg
160x10Assigned by compliance rule exceptionWithdraw exception
Never delete an XOrigin=8 assignment directly. Always unsubscribe through PersonWantsOrg. Direct deletion leaves the IT Shop request in an inconsistent state.

System Columns (all tables)

ColumnTypeMeaning
XObjectKeystringGlobally unique cross-table key — XML format above
XDateInserteddatetimeRow creation timestamp (set by OIM, not writable)
XDateUpdateddatetimeLast modification timestamp
XMarkedForDeletionint0 = active, 1 = soft-deleted (awaiting cleanup process)
XUserInsertedstringOIM user account that created the row
XUserUpdatedstringOIM user account that last modified the row

📋 Templates (TPL) #

Templates automatically provision target-system accounts (e.g., Active Directory) based on identity data. They define what to create and when.

Key Concepts

ConceptDescription
Account DefinitionLinks an OIM identity to a target-system account type. Controls who gets an account and under which conditions.
TemplateDefines default values for account properties (SamAccountName, OU, etc.) when creating or updating accounts.
Automatic Creation ModeControls if accounts are created automatically on assignment, manually, or for certain identity types only.
Mapping RuleA VB.NET inline expression transforming an identity property into a target-system property at creation time.
TemplateOverwriteControls whether the template re-applies on subsequent updates (overwrite) or only runs at initial creation.

VB.NET Expression Examples

' Build SamAccountName: FirstInitial + LastName, lowercase
Left($FirstName:String$, 1).ToLower() & $LastName:String$.ToLower()

' Display name: "LastName, FirstName"
$LastName:String$ & ", " & $FirstName:String$

' OU path based on department
"OU=" & $UID_Department_Display:String$ & ",OU=Users,DC=company,DC=com"

' Conditional OU: external vs internal
IIf($IsExternal:Bool$, "OU=External,DC=company,DC=com", "OU=Internal,DC=company,DC=com")

' Account move on department change — read new OU from department hierarchy
"OU=" & $Department_DepartmentName:String$ & ",OU=Users,DC=company,DC=com"
Template inheritance: Child account definitions inherit template rules from their parent. To override a specific rule in the child, add an explicit rule with the same property name — the child rule takes precedence.

⚙️ Processes (PRO) #

Processes are event-driven automation chains. When an object changes, the OIM event system enqueues a process chain in JobQueue for a Job Server to pick up.

ElementDescription
ProcessNamed chain of process steps triggered by an object event
Process StepIndividual action: run a script, call external program, send email, execute SQL
EventTrigger: Insert, Update, Delete, or custom named events
Process ParameterData passed from the triggering object to steps (e.g., $ObjectKeyBase, $UID_Person$)
JobQueuePending processes; Ready2EXE = 'TRUE' means available for pickup
Scheduled TriggerProcess chains can also be triggered on a cron-style schedule via the Scheduler object in Designer

Common VB.NET Script Step Patterns

' Fire a custom event from another process step
Dim obj As IEntity = Session.Source.GetSingle("Person", "UID_Person = '" & uid & "'")
obj.FireEvent("CCC_MyCustomEvent")
Session.Source.Save(obj)

' Read process parameters inside a script step
Dim objectKey As String = Value("$ObjectKeyBase$")
Dim uidPerson As String = Value("$UID_Person$")

' Conditional logic — abort process step
If condition Then
    RaiseError("Condition not met — aborting step")
End If
Enqueue from C# API code: Use u.GenerateAsync("ProcessName", new Dictionary<string,string> {{ "UID_Person", uid }}, ct) inside a UnitOfWork to programmatically enqueue a process chain.

DBQueue Processing (9.3)

In 9.3 the DBQueue Processor is a .NET service component — no SQL Agent job required. Monitor queue health in Job Queue Info. If provisioning is stalled, check: (1) DBQueue service running, (2) JobQueue.Ready2EXE = 'TRUE' rows exist, (3) Job Server service is alive, (4) server functions are correctly assigned.

🎯 Target Systems & Provisioning #

ConceptDescription
UNS RootRoot container for a target system namespace (one per connected system)
UNS AccountTarget-system user account linked to an OIM identity (Person)
Outstanding ObjectsObjects discovered in the target system but not yet reconciled into OIM's model
Full Managed ModeOIM fully controls the account — AD group membership is written by provisioning
Job GateA Job provider on one server + Job destination on another server; they communicate within the Jobgate configuration

Provisioning Chain — Step by Step

StepWhat Happens
1. Change committedA column value changes on a provisioning-tracked table (e.g., ADSAccount.IsLocked). OIM writes a row to JobQueue.
2. DBQueue Processor.NET component scans JobQueue for Ready2EXE='TRUE' entries and dispatches them to the appropriate Job Server queue.
3. Job Server picks upThe Job Server service polls its queue. It matches the process step's server function (e.g., ADSConnector) to decide which server handles it.
4. Connector executesThe connector (AD Connector, LDAP connector, etc.) translates the OIM change into a target system operation (LDAP modify, Graph API call, etc.).
5. ConfirmationOn success the process step is marked complete; on failure it enters an error state and may retry.
Troubleshooting stalled provisioning: (1) Is the DBQueue service running? (2) Does the JobQueue row have Ready2EXE='TRUE'? (3) Is the Job Server service alive and polling? (4) Does the Job Server have the correct server function assigned? (5) Is the target system reachable? (6) Are the Person and account both active (not IsInActive=1)?

🛒 IT Shop #

The IT Shop is OIM's self-service request portal. Users browse a service catalog and request access; requests flow through approval workflows before being provisioned.

Service Catalog Hierarchy

Shopping Center  (ITShopOrg, level=Center)
  └─ Shop        (ITShopOrg, level=Shop)
       └─ Shelf  (ITShopOrg, level=Shelf)  ← access-controlled per audience
            └─ Service Item (AccProduct)   ← the requestable product
ObjectTableNotes
Shopping CenterITShopOrgTop-level container; one per organisation or business unit
ShopITShopOrgGroups related shelves; controls which audience sees them
ShelfITShopOrgImmediate parent of service items; audience assigned here
Service ItemAccProductThe requestable entitlement (business role, application, account, etc.)

PersonWantsOrg — Request Lifecycle

Each IT Shop request is a row in PersonWantsOrg. The OrderState column tracks its lifecycle:

NotYetOrdered→ Request→ Pending→ Assigned
Assigned→ Unsubscribedor OrderCancelledor OrderFailed
OrderStateMeaning
NotYetOrderedItem added to cart but not yet submitted
RequestCart submitted; pre-approval checks running
PendingAwaiting approval decision(s)
AssignedFully approved and provisioned — access is live
UnsubscribedUser or admin called Unsubscribe; provisioning removal queued
OrderCancelledRequest was denied or cancelled during approval
OrderFailedTechnical provisioning failure after approval

ShoppingCartOrder.CheckStatus

ValueMeaning
0Unchecked — cart checks have not run yet
1OK — all checks passed, cart can be submitted
2Pending — ICartCheckProvider returned a pending status

Unsubscribe Pattern (C#)

// Find the active PersonWantsOrg row for this assignment
var q = Query.From("PersonWantsOrg")
    .Where($"ObjectKeyAssignment = '{objectKey}' AND OrderState = 'Assigned'")
    .OrderBy("XDateInserted desc").SelectAll();
var r = await qr.Session.Source().TryGetAsync(q, EntityLoadType.DelayedLogic, ct)
    .ConfigureAwait(false);
if (r.Success)
{
    await r.Result.CallMethodAsync("Unsubscribe", ct).ConfigureAwait(false);
    await r.Result.SaveAsync(qr.Session, ct).ConfigureAwait(false);
}

🏢 Role Model #

Three Role Types

TypeTablePurpose
Business RolesOrgMap company structures (projects, departments, groups). Members inherit all resources assigned to the role. Hierarchical — child roles inherit from parents.
Application RolesAERoleControl who can administer OIM itself — who can create business roles, who are attestors, who are role approvers. Built-in groups like vi_4_AUDITING_AUDITOR.
System RolesESetItUsedPackaged collections of company resources frequently assigned together. Assigned via IT Shop or directly. Not hierarchical.

Business Role Inheritance Engine

When a person is a member of a Business Role (PersonInOrg), OIM's inheritance engine walks the role hierarchy and grants all resources assigned to every ancestor role. The view PersonInBasetree (and its variants) materialises this inheritance. The resulting assignments get XOrigin = 2 (inherited).

Key distinction: PersonInOrg is the direct/requested membership table. PersonHasRole views represent the effective (inherited + direct) memberships. When writing queries to check if a person has effective access to a role, always query the view, not the base table.

vi_4_ATTESTATION_APPROVEREligible to make attestation decisions

🔑 Permissions (PMS) #

TypeDescription
Role-based permissionsAccess granted through Business Roles, Application Roles, or IT Shop requests
Permission GroupsOIM-internal groups (e.g., vi_4_AUDITING_AUDITOR) controlling API and portal access
Table permissionsColumn-level read/write control per permission group, enforced by the OIM object layer
Method permissionsAPI endpoints check permission groups before executing logic

Checking Permissions in API Code

// Check if current user is in a permission group
bool isAdmin = qr.Session.IsInRole("vi_4_ITSHOPADMIN_ADMIN");

// Restrict an endpoint to a permission group
builder.AddMethod(Method.Define("webportalplus/admin/action")
    .WithPermission("vi_4_ITSHOPADMIN_ADMIN")  // 401 if not in group
    .Handle<PostedData, object>("POST", async (body, qr, ct) => { ... }));
Permission layering: Object permission constraints applied via the OIM schema cannot be overridden by API code. WithWritableColumns() limits further but cannot grant more than the schema permits. The OIM session enforces schema-level column permissions before any API code runs.

✅ Attestations (ATT) #

Attestation (recertification) is the periodic review of existing access rights. Approvers confirm or revoke entitlements.

Key Tables

Table / ViewPurpose
AttestationCaseOne row per access right under review; holds decision state
AttestationPolicyDefines scope (what is attested), workflow, recertification interval
ATT_VAttestationDecisionPersonView listing eligible approvers for each open case — always check this before making a decision

AttestationPolicy Configuration (9.3)

FieldDescription
UID_PWODecisionMethodApproval workflow to use
NextApproveDateWhen the next attestation run is scheduled
IsApproveRequiredForAllAll approvers must decide, not just one
TermsOfUseFileNew in 9.3: attach a PDF file as terms of use for approvers to acknowledge

Making a Decision via API

// 1. Verify eligibility
string wc = $"XObjectKey = '{xsubkey}' AND UID_AttestationCase IN (" +
    $"SELECT UID_AttestationCase FROM ATT_VAttestationDecisionPerson " +
    $"WHERE UID_PersonHead = '{uidPerson}')";
bool isEligible = await qr.Session.Source()
    .ExistsAsync("AttestationCase", wc, ct).ConfigureAwait(false);
if (!isEligible) throw new InvalidOperationException("Not an eligible approver.");

// 2. Load the case
var q = Query.From("AttestationCase").SelectAll().Where($"XObjectKey = '{xsubkey}'");
var r = await qr.Session.Source().TryGetAsync(q, EntityLoadType.DelayedLogic, ct)
    .ConfigureAwait(false);

// 3. MakeDecision: (approverUID, approve=true/false, reason, justificationUID, subLevel=-1)
await r.Result.CallMethodAsync("MakeDecision", new object[]
    { uidPerson, true, reason, null, -1 }, ct).ConfigureAwait(false);
await r.Result.SaveAsync(qr.Session, ct).ConfigureAwait(false);

⚖️ Compliance & SoD #

Segregation of Duties (SoD) ensures no individual holds conflicting privileges that enable fraud or oversight bypass. OIM enforces SoD both preventively (at request time) and detectably (through attestation).

Compliance Rule Types

Rule TypeDescription
Entitlement ConflictDefines two sets of entitlements (roles/accounts) that must not be held simultaneously
Role-based ConflictApplies conflict detection at the Business Role level — role assignments trigger the rule
Condition-basedCustom VB.NET condition evaluated against identity attributes

Control Types

TypeWhen It RunsMechanism
PreventiveAt request submission in IT ShopICartCheckProvider returns error if request would create a violation; user cannot submit
DetectiveContinuously via calculation jobs; surfaced in attestationComplianceViolation rows created; attestation policy can scope to open violations

Reading Violations in API Code

// Find active SoD violations for a person
var q = Query.From("ComplianceViolation")
    .Select("UID_ComplianceRule", "IsException", "XDateInserted")
    .Where($"UID_Person = '{uidPerson}' AND IsException = 0");
var violations = await qr.Session.Source()
    .GetCollectionAsync(q, EntityLoadType.DelayedLogic, ct)
    .ConfigureAwait(false);

foreach (var v in violations)
{
    string ruleUid = v.GetValue<string>("UID_ComplianceRule");
    // ... report or act on it
}
Exception handling: When a violation has IsException=1, an authorised approver has explicitly approved the exception. Exception approvals are time-limited and appear in attestation for periodic review.

🔧 API Plugin Architecture #

OIM 9.3 exposes data via a RESTful CompositionAPI built on ASP.NET Core. Custom functionality is added by deploying plugin DLLs that the API Server discovers at startup.

Plugin Project Setup

ItemDetail
Target frameworknet8.0
Required referencesQBM.CompositionApi.dll, QER.CompositionApi.dll, VI.DB.dll, VI.Base.dll — from the OIM install dir. Do NOT reference QBM.CompositionApi.Server.dll in your plugin project.
DLL namingMust match *.CompositionApi.Server.Plugin.dll or *.CompositionApi.dll — the API Server only loads files matching these patterns.
Assembly attributeAdd [assembly: QBM.CompositionApi.PlugIns.Module("YourModuleName")] in AssemblyInfo.cs
DeploymentCopy to the API Server's plugin directory (or use Software Loader with bin\ prefix so files land in the correct sub-directory)

Minimal Plugin Registration

using QBM.CompositionApi.ApiManager;
using QBM.CompositionApi.Definition;

namespace QBM.CompositionApi
{
    public class MyPlugin : IPlugInMethodSetProvider
    {
        public IMethodSetProvider Create(IExtensibilityService extensibility)
        {
            var set = new MethodSet();
            set.AppId = "portal";   // target API project ("portal" = Web Portal)
            extensibility
                .FindAttributeBasedApiProviders<IApiProvider>()
                .ForEach(p => set.Register(p));
            return set;
        }
    }
}

Endpoint Class Pattern

// Each endpoint is a separate public class
[ApiProvider]
public class MyEndpoint : IApiProviderFor<PortalApiProject>, IApiProvider
{
    public void Build(IApiBuilder builder)
    {
        builder.AddMethod(Method.Define("webportalplus/myfeature/get")
            .HandleGet(async (qr, ct) =>
            {
                string uid = qr.Session.User().Uid;
                var q = Query.From("Person")
                    .Select("FirstName", "LastName")
                    .Where($"UID_Person = '{uid}'");
                var r = await qr.Session.Source()
                    .TryGetAsync(q, EntityLoadType.DelayedLogic, ct)
                    .ConfigureAwait(false);
                return r.Result;
            }));
    }
}

🧲 IBootstrapper vs IPlugInMethodSetProvider #

InterfaceMethodWhen it runsUse for
IBootstrapperRegisterServices(IPluginService)API Server initialisation — DI container setup phaseRegistering custom services, ICartCheckProvider, IRecommendationProvider, IPasswordItemProvider, IShopComponent
IPlugInMethodSetProviderCreate(IExtensibilityService)Routing initialisation phase — after DI is readyDefining API endpoints via MethodSet and Build() calls
// IBootstrapper — registers services before routing is built
public class MyBootstrapper : IBootstrapper
{
    public void RegisterServices(IPluginService service)
    {
        // Register IT Shop cart check
        service.GetPluginManager()
               .Register<ICartCheckProvider>(new MyCartCheckProvider());
        // Register attestation recommendation
        service.GetPluginManager()
               .Register<ATT.CompositionApi.Recommendation.IRecommendationProvider>(
                   new MyAttestationRecommendationProvider());
    }
}

// IPlugInMethodSetProvider — defines API routes
public class MyPlugin : IPlugInMethodSetProvider
{
    public IMethodSetProvider Create(IExtensibilityService ext)
    {
        var set = new MethodSet { AppId = "portal" };
        ext.FindAttributeBasedApiProviders<IApiProvider>()
           .ForEach(p => set.Register(p));
        return set;
    }
}
One DLL can implement both. A single plugin project typically has one class for IBootstrapper (services) and one for IPlugInMethodSetProvider (endpoints).

📥 Bulk Actions (024_002) #

Bulk Actions let the Web Portal process CSV files row-by-row against custom API endpoints. The portal calls four endpoints in a fixed sequence.

1 · startaction

/startaction

Pre-flight. Receives header names + total row count. Returns { message, permission, ActionAttributes }. Creates shared state (e.g., a ShoppingCartOrder). ActionAttributes is forwarded to all subsequent calls.

2 · validate

/startValidate → /validate

Called once per row. Returns object[] with { column } for valid columns or { column, errorMsg } for invalid ones. Does not write to DB. May add ValidationAttributes per row.

3 · action

/action

Called once per valid row. Performs the actual DB write. Receives all column values + ActionAttributes + per-row ValidationAttributes.

4 · endaction

/endaction

Post-flight. Receives import stats (successCount, failCount, errorRows). Returns a summary message string. Clean up shared state here.

startaction — Create Shared State

Generate Unique Central Account (Fixed)

v1 bug fixed: The original loop was missing Exit Do, causing all 10 iterations to always run even after finding a free account. Corrected below.
Public Function CCC_Generate_CentralAccount() As String
    Dim f As ISqlFormatter = Session.SqlFormatter
    Dim maxAttempts As Integer = 10
    Dim attempt As Integer = 0
    Dim random As New Random()
    Dim valueToReturn As String = String.Empty

    Do
        attempt += 1
        Dim randomNumber As Integer = random.Next(10000, 99999)
        Dim centralAccount As String = "T" & randomNumber.ToString()

        ' ISqlFormatter prevents SQL injection
        Dim wc As String = f.Comparison("CentralAccount", centralAccount, ValType.String, CompareOperator.Equal)
        Dim isTaken As Boolean = Session.Source.GetCount(
            Query.From("Person").Where(wc).Select("CentralAccount")) > 0

        If Not isTaken Then
            valueToReturn = centralAccount
            Exit Do   ' ← FIX: stop as soon as a free account is found
        End If
    Loop While attempt < maxAttempts

    Return valueToReturn
End Function

    ' Strip trailing digit if present (handles re-run scenarios)

            Query.From("Person").Where(wc).Select("CustomProperty03"))

End If

        [Parameter(Mandatory)][string]$Server,
        [Parameter(Mandatory)][string]$Database,

        [switch]$Integrated

function Get-OIMPersons {
    param([string]$ConnectionString, [string]$UID_Person = $null)
    $conn = New-Object System.Data.SqlClient.SqlConnection $ConnectionString
    $cmd  = $conn.CreateCommand()
    $cmd.CommandText = @"
SELECT UID_Person, FirstName, LastName, CentralAccount
FROM dbo.Person
WHERE (@UID IS NULL OR UID_Person = @UID) AND IsInActive = 0
"@
    $cmd.Parameters.AddWithValue("@UID",
        [object]([string]::IsNullOrEmpty($UID_Person) ? [DBNull]::Value : $UID_Person)) | Out-Null

    try {
        $conn.Open()
        $reader  = $cmd.ExecuteReader()
        $idxUID  = $reader.GetOrdinal("UID_Person")
        $idxFN   = $reader.GetOrdinal("FirstName")
        $idxLN   = $reader.GetOrdinal("LastName")
        $results = [System.Collections.Generic.List[PSCustomObject]]::new()
        while ($reader.Read()) {
            $results.Add([PSCustomObject]@{
                UID_Person    = $reader.GetValue($idxUID)
                FirstName     = $reader.GetValue($idxFN)
                LastName      = $reader.GetValue($idxLN)
            })
        }
        return $results
    }
    finally {
        if ($reader) { $reader.Close() }
        if ($conn)   { $conn.Close()   }
    }
}

🔧 Troubleshooting Guide #

Symptom / ErrorCauseFix
ApiServer0037 — duplicate route Two endpoints registered with the same URL pattern Check all Method.Define("...") calls for duplicate paths. Remember the URL is case-insensitive.
Plugin DLL not loaded — endpoints return 404 DLL name doesn't match *.CompositionApi.Server.Plugin.dll pattern, or it's in the wrong directory Rename the output DLL. Check API Server log for "Loading plugin" entries. Verify deploy directory. Check [assembly: Module] attribute.
HTTP 400 on every POST — "XSRF token missing" Angular client not sending X-XSRF-TOKEN header, or cookie/header mismatch after session restore Read XSRF-TOKEN cookie after login and inject it as a header. Clear browser cookies and re-login if stale.
Provisioning stuck — account changes don't appear in AD DBQueue not processing, Job Server offline, or missing server function Check: (1) DBQueue service running; (2) JobQueue.Ready2EXE='TRUE' rows exist; (3) Job Server service alive; (4) correct server function assigned to Job Server; (5) AD connector credentials valid.
Bulk action session expired mid-import Long-running import exceeds session timeout; each action call re-validates the session Reduce CSV row count per batch, or increase session timeout in imxweb.json (SessionTimeout). Ensure the Angular client sends keepalive pings.
Transport import conflict — "Object already exists" Two transport packages contain conflicting versions of the same object (same UID, different change labels) Consolidate into one change label, or import in the correct order. Use the Deployment Manager to manage dependency ordering.
TryGetAsync returns r.Success = false No row matched the Where clause, or the user's session lacks read permission on that table Verify the Where clause (log the query string). Check table-level read permission for the session user's permission groups.
VB.NET script fires on every save, not just creation Missing old-value guard — script runs unconditionally Wrap logic in If DbVal.IsEmpty($Column[o]:String$, ValType.String) Then to restrict to first-time assignment.
Missing XSRF token when calling from custom Angular component IMX Angular library intercepts HTTP — custom HttpClient bypasses this Use the OIM-provided Angular service (imx-api) for API calls, or manually attach the token interceptor to your HttpClient provider.
