🏗️ Architecture

OIM 9.3 is composed of several tiers that interact through a central SQL database and a service bus.

ComponentRole
JobServerExecutes process chains, handles provisioning events, runs scheduled jobs
API ServerASP.NET Core host serving the CompositionAPI. Plugins extend it via DLLs
Web PortalAngular SPA consuming the API Server. Renders UI for requests, attestations, reports
Sync EngineSynchronizes external target system data into OIM's internal model
DesignerWindows client for configuring schemas, templates, processes, scripts
SQL DatabaseCentral data store. All objects, logs, and configuration reside here
Namespace convention All custom plugin code uses the QBM.CompositionApi namespace and references DLLs from C:\Program Files\One Identity\One Identity Manager\. Projects target .NET 8.0.

📋 Templates (TPL)

Templates in OIM automatically provision target-system accounts (e.g., Active Directory) based on identity data. They define what to create and when.

Key Concepts

ConceptDescription
Account DefinitionThe rule that links an OIM identity to a target-system account type. Controls who gets an account and under which conditions.
TemplateDefines the default values for account properties (e.g., SamAccountName, OU) when creating or updating accounts.
Automatic Creation ModeDetermines if accounts are created automatically (on assignment), manually, or only for certain identity types.
Mapping RuleA VB.NET inline expression that transforms an identity property into a target-system property value at creation time.

Template VB.NET Expression Examples

Template rules use OIM's inline VB.NET syntax. The $Column:Type$ syntax reads property values from the current entity.

' Build SamAccountName as FirstInitial + LastName, lowercase
Left($FirstName:String$, 1).ToLower() & $LastName:String$.ToLower()

' Build display name as "LastName, FirstName"
$LastName:String$ & ", " & $FirstName:String$

' Build OU path based on department
"OU=" & $UID_Department_Display:String$ & ",OU=Users,DC=company,DC=com"

' Conditional: external users go to a separate OU
IIf($IsExternal:Bool$, "OU=External,DC=company,DC=com", "OU=Internal,DC=company,DC=com")

Template Exercises (TPL-1 through TPL-15)

The Academy provides 15 hands-on exercises covering: basic account creation, naming conventions, multi-domain scenarios, account move on department change, template inheritance, and overwrite behaviour.

Key exercise patterns: TPL-11 through TPL-15 cover advanced template queries — including reading parent-department hierarchies, computing availability dates, and building dynamic OU paths from multi-level organisational data.

⚙️ Processes (PRO)

Processes in OIM are event-driven automation chains. When an object changes (e.g., a Person is created, an account is disabled), the OIM event system fires a process chain on the Job Server.

Process Structure

ElementDescription
ProcessA named chain of process steps triggered by an object event
Process StepAn individual action: run a script, call an external program, send an email, execute SQL
EventThe trigger: Insert, Update, Delete, or custom named events
Process ParameterData passed from the triggering object to the process steps (e.g., $ObjectKeyBase)
Job QueuePending processes waiting to be picked up by a Job Server instance

Common Process Patterns

' Example: Fire a custom event on a Person
Dim obj As IEntity = Session.Source.GetSingle("Person", "UID_Person = '" & uid & "'")
obj.FireEvent("CCC_MyCustomEvent")
Session.Source.Save(obj)

' Example: Read process parameters inside a script step
Dim objectKey As String = Value("$ObjectKeyBase$")
Dim uidPerson As String = Value("$UID_Person$")
Process generator — The u.GenerateAsync("ProcessName", parameters, ct) API call (seen in DataExplorer+ endpoints) enqueues a process directly from C# API code.

🎯 Target Systems

Target systems are the external directories and applications OIM provisions accounts into: Active Directory, LDAP, SAP, Azure AD, Exchange, and custom systems via connectors.

ConceptDescription
UNS RootThe root container for a target system namespace (one per connected system)
UNS AccountA target-system user account linked to an OIM identity (Person)
Outstanding ObjectsObjects discovered in the target system but not yet reconciled into OIM's model
ProvisioningThe Job Server process that writes OIM data changes back to the target system
XOriginBitmask tracking the source of a membership assignment: 1 = direct, 8 = requested via IT Shop
XOrigin bitmask in code:
XOrigin & 1 = 1 → directly assigned (can be marked for deletion)
XOrigin & 8 = 8 → requested via IT Shop (must be unsubscribed via PersonWantsOrg)

🔌 Connectors & Sync Editor

The Sync Editor configures the bidirectional synchronisation between OIM and target systems. It defines mappings, rules, and scopes.

Synchronisation Flow

PhaseDescription
ReadConnector reads all objects from the target system into a staging buffer
CompareSync Engine compares staged data against OIM's current state
ClassifyDetermines which objects are new, changed, deleted, or outstanding
ProvisionWrites OIM changes back to the target system via connector

Key Configuration Objects

ObjectPurpose
Synchronisation ProjectContainer for all settings related to one connected system
SchemaThe metadata describing object classes and attributes available in the target
MappingMaps target-system attributes to OIM columns with optional transformation rules
ScopeLimits which objects are included in a sync run (e.g., specific OUs)
Variable SetEnvironment-specific parameters that override mappings without touching the project
Outstanding objects — Objects found in the target system but not yet assigned to an OIM identity appear as outstanding. They must be explicitly linked or deleted to resolve the discrepancy.

🔀 Approval Workflows (WFL)

Workflows in OIM control who must approve requests, role assignments, and attestation decisions before they are processed.

ConceptDescription
WorkflowA named sequence of approval levels assigned to a service item or attestation policy
Approval LevelA step in the workflow; defines approvers and timeout behaviour
Approval StepA single approver within a level: a person, a role, a dynamic resolver, or an automated decision
Decision TypeApprove, Deny, Delegate, Escalate, Reroute
Escalation RuleAutomatically escalates to the next approver if no decision is made within N days

Workflow Categories

Workflow TypeTrigger
IT Shop WorkflowService item requested via Shopping Cart
Attestation WorkflowAttestation run created for a policy
Entitlement WorkflowRole / business role assignment submitted for approval

🔑 Permissions (PMS)

OIM uses a permission group model to control access to the Designer, portal, and database objects.

TypeDescription
Role-based permissionsAccess granted through Business Roles, Application Roles, or IT Shop requests
Permission GroupsOIM-internal groups (e.g., vi_4_AUDITING_AUDITOR) that control API and portal access
Table permissionsColumn-level read/write control per permission group, enforced by the OIM object layer
Method permissionsAPI endpoints check permission groups before executing logic
Important: Object permission constraints applied via the OIM schema cannot be overridden by API code. The WithWritableColumns() call limits further but cannot exceed the permission layer.

✅ Attestations (ATT)

Attestation (recertification) is the periodic review of existing access rights. Approvers confirm or revoke entitlements.

Key Tables

TablePurpose
AttestationCaseA single attestation decision item (one row per access right being reviewed)
AttestationPolicyDefines what objects are attested, workflow used, and recertification schedule
ATT_VAttestationDecisionPersonView listing eligible approvers for each open case

Making a Decision via API (MakeDecision)

// Load the attestation case
var query = Query.From("AttestationCase").SelectAll().Where($"XObjectKey = '{xsubkey}'");
var tryget = await qr.Session.Source().TryGetAsync(query, EntityLoadType.DelayedLogic, ct)
    .ConfigureAwait(false);
IEntity attestationCase = tryget.Result;

// Call the OIM built-in MakeDecision method
await attestationCase.CallMethodAsync("MakeDecision", new object[]
{
    uidPerson,   // approver UID
    true,        // true = Approve, false = Deny
    reason,      // reason text from UI pop-up
    null,        // UidJustification (optional)
    -1           // SubLevel (-1 = default)
}, ct).ConfigureAwait(false);

await attestationCase.SaveAsync(qr.Session, ct).ConfigureAwait(false);

Eligibility Check Pattern

// Ensure the current user is an eligible approver before making a decision
string wc = string.Format(
    "XObjectKey = '{0}' AND UID_AttestationCase IN (" +
    "SELECT UID_AttestationCase FROM ATT_VAttestationDecisionPerson " +
    "WHERE UID_PersonHead = '{1}')", xsubkey, uidPerson);

bool isEligible = await qr.Session.Source()
    .ExistsAsync("AttestationCase", wc, ct).ConfigureAwait(false);

if (!isEligible)
    throw new InvalidOperationException("You are not the eligible approver.");

📦 Transport (TRS)

OIM transports package configuration objects (templates, processes, scripts, custom properties) into a ZIP file for deployment across environments.

ComponentDescription
Transport PackageZIP file containing serialised OIM objects as XML
Change LabelGroups related changes; a transport is built from one or more change labels
ImportApplies objects from a transport to the target environment via the Designer import wizard
Deployment ManagerThird-party tool (Intragen DM) for scripted transport management — see § Deployment Manager
Transport hygiene: Always transport with a dedicated change label per feature. Mixing unrelated changes in one label makes rollback difficult and can cause dependency issues during import.

🔧 API Plugin Architecture

OIM 9.3 exposes its data via a RESTful CompositionAPI built on ASP.NET Core. Custom functionality is added by deploying plugin DLLs that the API Server discovers and loads at startup.

Plugin Project Setup

FilePurpose
*.csprojTarget net8.0; reference QBM.CompositionApi.dll, QER.CompositionApi.dll, VI.DB.dll, VI.Base.dll
*.CompositionApi.Server.Plugin.dllOutput DLL pattern — the API Server only loads files matching *.CompositionApi.Server.dll or *.CompositionApi.dll
Plugin entry pointImplements IPlugInMethodSetProvider → creates a MethodSet with an AppId and wires up providers via IExtensibilityService.FindAttributeBasedApiProviders<T>()

Minimal Plugin Registration

using QBM.CompositionApi.ApiManager;
using QBM.CompositionApi.Definition;

namespace QBM.CompositionApi
{
    public class MyPlugin : IPlugInMethodSetProvider
    {
        public IMethodSetProvider Create(IExtensibilityService extensibility)
        {
            var set = new MethodSet();
            set.AppId = "portal";   // target API project
            extensibility.FindAttributeBasedApiProviders<IApiProvider>()
                         .ForEach(p => set.Register(p));
            return set;
        }
    }
}

API Provider Pattern

Each endpoint is a separate public class implementing IApiProvider and IApiProviderFor<TProject>.

public class MyEndpoint : IApiProviderFor<PortalApiProject>, IApiProvider
{
    public void Build(IApiBuilder builder)
    {
        builder.AddMethod(Method.Define("webportalplus/myfeature/get")
            .HandleGet(async (qr, ct) =>
            {
                string uid = qr.Session.User().Uid;
                var q = Query.From("Person").Select("FirstName", "LastName")
                             .Where($"UID_Person = '{uid}'");
                var result = await qr.Session.Source()
                    .TryGetAsync(q, EntityLoadType.DelayedLogic, ct)
                    .ConfigureAwait(false);
                return result.Result;
            }));
    }
}

🔄 API Server Lifecycle

Understanding the startup phases helps with debugging plugin load failures.

PhaseDescription
ASP.NET Core InitBasic services configured. Plugins implementing IBuilderPlugIn run here.
API InitializationPlugins implementing IPlugIn add their API services.
Routing InitAPI projects are discovered; IPlugInMethodSetProvider is called. All Build() methods run.
API ValidationConsistency checks run. Errors here cause 404s on affected projects but don't stop the server.
StartServer responds to requests.

Request Processing Flow

StepAction
1ASP.NET Core matches the URL to a route
2API Server associates request with a session group (imx_sessiongroup cookie)
3XSRF token validated
4Language preferences applied to async context
5Validators run (from .WithValidation())
6Handler code executes
Session management: OIM does NOT use ASP.NET Core session. Each API project in a session group has an independent Identity Manager session. Use qr.Session.User().Uid to get the authenticated user's UID.

📚 Entity Methods

Entity methods integrate directly with the OIM object layer, automatically generating REST endpoints for reading, creating, updating, and deleting database objects.

CRUD Operations

// Read-only with specific columns
Method.Define("persons")
    .FromTable("Person")
    .EnableRead()
    .WithResultColumns("FirstName", "LastName", "CentralAccount")

// Full CRUD with writable columns
Method.Define("persons/edit")
    .FromTable("Person")
    .EnableRead()
    .EnableCreate()
    .EnableUpdate()
    .WithWritableColumns("FirstName", "LastName", "UID_Department")
    .EnableDelete()
    .WithWhereClause("IsInActive = 0")   // base filter applied to all ops

// Conditional writability
.WithWritableColumns("Phone", e => e.GetValue<bool>("IsExternal"))

// Interactive entity (recommended when UI interactions and permissions matter)
.As(EntityType.Interactive)

Filters

Filters allow the Angular client to offer filter options in the UI. Always enable DataModelApi when using filters.

// Static filter (fixed set of options, emitted to OpenAPI)
.With(m =>
{
    m.EnableDataModelApi = true;
    m.OptionalClauseProviders.Add(new StateFilter());
})

private class StateFilter : IPredefinedFilter
{
    public string UrlParameterName => "state";
    public IDictionary<string, IPredefinedFilterOption> Options =>
        new Dictionary<string, IPredefinedFilterOption>
        {
            { "active",   new PredefinedFilterOption { ClauseProvider = new ClauseProvider("IsInActive=0") } },
            { "inactive", new PredefinedFilterOption { ClauseProvider = new ClauseProvider("IsInActive=1") } }
        };
}

// Dynamic filter (options calculated at runtime)
m.DynamicFilters.Add(new DepartmentFilter());

// Filter provider (9.3+) — virtual filterable property
var mod = builder.Resolver.Resolve<IModifierService>();
mod.RegisterFilterProvider("Person", new ReportingToFilterProvider());

Grouping

Method.Define("persons")
    .FromTable("Person")
    .EnableRead()
    .EnableGrouping("UID_Department")   // groups by department
    .With(m => {
        m.EnableDataModelApi = true;
        m.EnableGroupingApi = true;
    });

Property Types

Use .WithCalculatedProperties() to attach derived data to entities returned by an entity method.

// FK property: add the department manager's UID to each Person
.WithCalculatedProperties(new FkProperty(
    "DepartmentManager", "Department", "UID_PersonHead", "UID_Department"))

// Child relation: list each person's direct reports
.WithCalculatedProperties(new CrProperty("Reports", "Person", "UID_PersonHead"))

// Many-to-many: list group memberships
.WithCalculatedProperties(new MrProperty("Groups", "ADSAccountInADSGroup", "UID_ADSAccount"))

// Calculated from code
.WithCalculatedProperties(new CalculatedProperty<string>(
    "FullName",
    ctx => $"{ctx.Entity.GetValue("FirstName")} {ctx.Entity.GetValue("LastName")}"))

// Enum property (maps integer to display string)
.WithCalculatedProperties(new EnumProperty("StatusDisplay", "IsInActive",
    new[] { new EnumValue(0, "Active"), new EnumValue(1, "Inactive") }))

Extended Data

Extended data allows supplementary data to travel alongside entity API responses — useful for additional context that doesn't fit neatly as entity properties.

// Read-only extended data provider
builder.AddMethod(Method.Define("persons/withextra")
    .FromTable("Person").EnableRead()
    .WithExtendedData(new MyExtendedDataProvider()));

private class MyExtendedDataProvider : IReadOnlyExtendedDataProvider<MyExtraData>
{
    public async Task<MyExtraData> GetExtendedDataAsync(
        IReadOnlyList<IEntity> entities, IRequest request, CancellationToken ct = default)
    {
        return new MyExtraData { Count = entities.Count, Timestamp = DateTime.UtcNow };
    }
}
public class MyExtraData { public int Count { get; set; } public DateTime Timestamp { get; set; } }

⚡ Custom HTTP Methods

When entity methods don't fit, use .Handle<TInput, TOutput>() for fully custom POST/GET handlers. This is the pattern used by all Bulk Action and DataExplorer+ endpoints.

// Custom POST — typed request/response
builder.AddMethod(Method.Define("webportalplus/myaction")
    .Handle<MyRequest, MyResponse>("POST", async (body, qr, ct) =>
    {
        // Access session and DB
        string uid = qr.Session.User().Uid;

        // Read from DB
        var q = Query.From("Person").Select("FirstName").Where($"UID_Person = '{uid}'");
        var r = await qr.Session.Source()
            .TryGetAsync(q, EntityLoadType.DelayedLogic, ct).ConfigureAwait(false);
        string name = r.Result.GetValue<string>("FirstName");

        // Write to DB
        using var u = qr.Session.StartUnitOfWork();
        var entity = await qr.Session.Source()
            .CreateNewAsync("MyTable", new EntityParameters { CreationType = EntityCreationType.DelayedLogic }, ct)
            .ConfigureAwait(false);
        await entity.PutValueAsync("SomeColumn", body.Value, ct).ConfigureAwait(false);
        await u.PutAsync(entity, ct).ConfigureAwait(false);
        await u.CommitAsync(ct).ConfigureAwait(false);

        return new MyResponse { Success = true, Name = name };
    }));

public class MyRequest  { public string Value { get; set; } }
public class MyResponse { public bool Success { get; set; } public string Name { get; set; } }

Session Helpers Reference

CallPurpose
qr.Session.User().UidUID of the currently authenticated user
qr.Session.Source().TryGetAsync(q, …)Read one entity; returns TryResult<IEntity>
qr.Session.Source().GetCollectionAsync(q, …)Read multiple entities
qr.Session.Source().ExistsAsync(table, wc, ct)Check if a row exists (fast, no entity load)
qr.Session.Source().CreateNewAsync(table, params, ct)Create a new entity in memory
qr.Session.StartUnitOfWork()Begin a write transaction (use using)
u.PutAsync(entity, ct)Stage an entity for write
u.CommitAsync(ct)Commit the transaction to the DB
entity.MarkForDeletion()Mark entity for delete (stage with u.PutAsync then CommitAsync)
entity.CallMethodAsync("MethodName", args, ct)Call an OIM built-in method (e.g., MakeDecision, Unsubscribe)
entity.SaveAsync(qr.Session, ct)Save entity directly (without explicit UoW)
u.GenerateAsync("ProcessName", params, ct)Enqueue a process chain
qr.Session.Config().GetConfigParmAsync("Custom\\…", ct)Read a configuration parameter
qr.Session.Resolve<IStatementRunner>()Execute raw SQL / stored procedures
ConfigureAwait(false) — Always append .ConfigureAwait(false) to every await call inside API handlers. This avoids deadlocks in the ASP.NET Core synchronisation context.
String interpolation vs string.Format — When building strings with \n line breaks, use $"text\nnewline", NOT string.Format(@"text\nnewline"). The verbatim prefix on Format produces \\n instead of an actual newline.

📥 Bulk Actions (024_002)

Bulk Actions allow the Web Portal to process CSV files row-by-row against custom API endpoints. The portal calls the four endpoints in a fixed sequence.

1 · startaction

/startaction

Pre-flight. Receives header names and total row count. Returns { message, permission, collectImportData? }. Creates shared state (e.g., a shopping cart) that action steps will use.

2 · validate

/startValidate → /validate

Called once per row. Returns object[] with { column } for valid columns or { column, errorMsg } for invalid ones. Does NOT write to the DB.

3 · action

/action

Called once per row (only for valid rows). Performs the actual DB write: create, update, or delete. Receives all column values for the current row.

4 · endaction

/endaction

Post-flight. Receives import statistics (success/failure counts). Returns a summary message. Clean up shared state here.

startaction — Creating a Shopping Cart

public class addEntitlementStartAction : IApiProviderFor<PortalApiProject>, IApiProvider
{
    public void Build(IApiBuilder builder)
    {
        builder.AddMethod(Method.Define("webportalplus/addentitlement/startaction")
            .Handle<PostedID, object>("POST", async (posted, qr, ct) =>
            {
                var loggedinuser = qr.Session.User().Uid;

                // Create a shopping cart order to hold all row actions
                var newID = await qr.Session.Source().CreateNewAsync(
                    "ShoppingCartOrder",
                    new EntityParameters { CreationType = EntityCreationType.DelayedLogic }, ct)
                    .ConfigureAwait(false);
                await newID.PutValueAsync("UID_Person", loggedinuser, ct).ConfigureAwait(false);
                await newID.PutValueAsync("CheckStatus", 2, ct).ConfigureAwait(false);

                string shoppingCartUid = await newID.GetValueAsync<string>("UID_ShoppingCartOrder")
                    .ConfigureAwait(false);

                using (var u = qr.Session.StartUnitOfWork())
                {
                    await u.PutAsync(newID, ct).ConfigureAwait(false);
                    await u.CommitAsync(ct).ConfigureAwait(false);
                }

                return new Dictionary<string, object>
                {
                    { "message",  $"Assigning {posted.totalRows} entitlements." },
                    { "permission", true },
                    // Pass the cart UID forward to subsequent action calls
                    { "ActionAttributes", new { shoppingCart = shoppingCartUid } }
                };
            }));
    }
    public class PostedID { public string[] headerNames { get; set; } public int totalRows { get; set; } }
}

validate — Per-Row Validation

// Validate that the posted PersonUID exists
var exists = await qr.Session.Source()

Generate Unique Central Account

Purpose: Generates a unique T##### central account identifier for new identities.

Public Function CCC_Generate_CentralAccount() As String
    Dim maxAttempts As Integer = 10
    Dim attempt As Integer = 0
    Dim random As New Random()
    Dim existingAccount As String
    Dim valueToReturn As String

    Do
        attempt += 1
        Dim randomNumber As Integer = random.Next(10000, 99999)
        Dim centralAccount As String = "T" & randomNumber.ToString()
        Dim whereClause As String = String.Format("CentralAccount = '{0}'", centralAccount)

        ' Returns False if no matching record exists (i.e., account is free)
        If Not Session.Source.TryGetSingleValue("Person", "CentralAccount", whereClause, existingAccount) Then
            valueToReturn = centralAccount
        End If
    Loop While attempt < maxAttempts

    Return valueToReturn
End Function

Unique Custom Property with Counter Suffix

Purpose: Ensures uniqueness of CustomProperty03 by appending a numeric suffix (value → value1 → value2 …).

' Guard: only runs when the OLD value was empty (first-time assignment)
If DbVal.IsEmpty($CustomProperty03[o]:String$, ValType.String) Then
    Dim f As ISqlFormatter = Session.SqlFormatter
    Dim lastName As String = $CustomProperty03:String$
    Dim lastChar As Char = lastName(lastName.Length - 1)
    Dim IsLastCharNumeric As Boolean = Char.IsDigit(lastChar)
    Dim lastNameLikeQuery As String = lastName & "%"
    Dim lastNameToInsert As String = lastName
    Dim counter As Integer = 1

    If IsLastCharNumeric Then
        lastName = lastName.Substring(0, lastName.Length - 1)
        lastNameLikeQuery = lastName & "%"
        lastNameToInsert = lastName
    End If

    Dim matching = Query.From("Person") _
        .Where(f.Comparison("CustomProperty03", lastNameLikeQuery, ValType.String, CompareOperator.like)) _
        .OrderBy("CustomProperty03").Select("CustomProperty03")
    Dim col As IEntityCollection = Session.Source.GetCollection(matching)

    For Each entry As IEntity In col
        Dim check = Query.From("Person") _
            .Where(f.Comparison("CustomProperty03", lastNameToInsert, ValType.String, CompareOperator.Equal)) _
            .Select("CustomProperty03")
        If Session.Source.GetCount(check) = 0 Then Exit For
        lastNameToInsert = lastName + counter.ToString
        counter += 1
    Next
    value = lastNameToInsert
End If

Connection String Builder

function New-SqlConnectionString {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)][string]$Server,
        [Parameter(Mandatory)][string]$Database,
        [pscredential]$Credential,
        [switch]$Integrated
    )
    if ($Integrated) {
        return "Server=$Server;Database=$Database;Integrated Security=True;" +
               "Encrypt=True;TrustServerCertificate=True;Connection Timeout=30;"
    }
    $u = $Credential.UserName
    $p = $Credential.GetNetworkCredential().Password
    return "Server=$Server;Database=$Database;User ID=$u;Password=$p;" +
           "Encrypt=True;TrustServerCertificate=True;Connection Timeout=30;"
}

Parameterized SQL Query Pattern

function Get-Users {
    param([string]$ConnectionString, [switch]$AskUserID)
    $conn   = New-Object System.Data.SqlClient.SqlConnection $ConnectionString
    $cmd    = $conn.CreateCommand()
    $cmd.CommandText = "SELECT UserID, UserName FROM dbo.[User] WHERE (@UserID IS NULL OR UserID = @UserID)"
    $userID = if ($AskUserID) { Read-Host "Enter UserID (or blank for all)" } else { $null }
    $cmd.Parameters.AddWithValue("@UserID", [object]($userID -eq '' ? [DBNull]::Value : $userID)) | Out-Null

    try {
        $conn.Open()
        $reader = $cmd.ExecuteReader()
        # Cache ordinals for performance
        $idxID   = $reader.GetOrdinal("UserID")
        $idxName = $reader.GetOrdinal("UserName")
        $results = [System.Collections.Generic.List[PSCustomObject]]::new()
        while ($reader.Read()) {
            $results.Add([PSCustomObject]@{
                UserID   = $reader.GetValue($idxID)
                UserName = $reader.GetValue($idxName)
            })
        }
        return $results
    }
    finally {
        if ($reader)  { $reader.Close() }
        if ($conn)    { $conn.Close()   }
    }
}

Structured Logging Pattern

# Logger factory — creates a per-run log file
function New-Logger {
    param([string]$FunctionName, [string]$LogPath)
    $timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
    $logFile   = Join-Path $LogPath "${FunctionName}_${timestamp}.log"
    return @{
        Info  = { param($msg) $line = "[INFO]  $msg"; Write-Host $line; Add-Content $logFile $line }
        Debug = { param($msg) $line = "[DEBUG] $msg"; Write-Host $line; Add-Content $logFile $line }
        Trace = { param($msg) $line = "[TRACE] $msg"; Write-Verbose $line; Add-Content $logFile $line }
        Error = { param($msg) $line = "[ERROR] $msg"; Write-Warning $line; Add-Content $logFile $line }
    }
}

Authentication with GUI Retry

function Get-ConnectionStringWithRetry {
    param([string]$Server, [string]$Database)
    $maxRetries = 3
    for ($i = 0; $i -lt $maxRetries; $i++) {
        $cred = $Host.UI.PromptForCredential("SQL Login", "Enter credentials", "", "")
        if ($null -eq $cred) { return $null }   # user cancelled

        $cs = New-SqlConnectionString -Server $Server -Database $Database -Credential $cred
        try {
            $test = New-Object System.Data.SqlClient.SqlConnection $cs
            $test.Open(); $test.Close()
            return $cs   # success
        }
        catch { Write-Warning "Login failed ($($_.Exception.Message)). Attempt $($i+1)/$maxRetries." }
    }
    throw "Authentication failed after $maxRetries attempts."
}
