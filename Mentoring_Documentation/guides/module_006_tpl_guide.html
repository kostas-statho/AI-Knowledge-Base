Templates

Code: TPL | Beginner–Intermediate | 15 exercises | ~4 days

Overview

Module Purpose

  • OIM templates are VB.NET scripts that auto-calculate column values
  • Progress from basic overwrite behaviour through date calculations
  • Cross-table lookups and multi-value property management
  • 15 exercises covering all major template patterns

Prerequisites

  • BAS module complete
  • Basic VB.NET knowledge (If/Else, String, date arithmetic)
  • OIM Designer access
  • OIM Manager and Object Browser access

Core Concepts

  • TC: Target Column — what the template WRITES TO
  • SC: Source Column — what the template READS FROM
  • Overwrite ON: Recalculates even if TC has existing data
  • Overwrite OFF: Only runs when TC is empty
  • Triggers: Insert, Update, Save (configurable)
  • MultiValueProperty: Column holding a list of values

VB.NET Reference

  • Value("Column") — reads saved value from same object
  • Value("Table","Col","uid") — reads from related table
  • DateAdd(DateInterval.Month, 2, date) — date arithmetic
  • MultiValueProperty exercises (TPL-12 to 15) — do in order

TPL-7

ExitDate = EntryDate + 2 Months (with Validation)

Requirements

Template on ExitDate. Calculated value = EntryDate + 2 months. Overwrite only if the current ExitDate is less than EntryDate + 2 months. If ExitDate is already beyond the 2-month threshold, keep it.

Logic Explained
Date Boundary Logic The threshold is DateAdd(DateInterval.Month, 2, EntryDate). If ExitDate < threshold → overwrite with threshold. If ExitDate >= threshold → keep the manually set value. This prevents OIM from "pulling back" a far-future exit date.
Test Cases
#Test ScenarioStepsExpectedStatus
1Set EntryDateSet a new EntryDate on the identityExitDate = EntryDate + 2 monthsPASS
2Set ExitDate to EntryDate minus 1 monthSet ExitDate to a date BEFORE the 2-month thresholdExitDate is overwritten back to EntryDate + 2 monthsPASS
3Set ExitDate to EntryDate + 1 monthSet ExitDate to 1 month after EntryDate (within the 2-month range)ExitDate takes the inserted value (within the allowed range)PASS
4Set ExitDate to EntryDate + 3 monthsSet ExitDate to 3 months after EntryDate (beyond threshold)ExitDate = EntryDate + 2 months (overwritten back — exceeds the maximum)PASS
Mentor Note Walk through the date arithmetic on a whiteboard first. EntryDate + 2 months = the boundary. Below boundary → overwrite. Above → keep. Trainees often confuse < and > here — the whiteboard prevents errors.

TPL-10

Unique Identifier with Numeric Suffix — CustomProperty03

Requirements

Template on CustomProperty03 — generates a unique identifier. If "BAS" already exists → use "BAS1". If "BAS1" already exists → use "BAS2". Continues incrementing until a free suffix is found.

Logic
Uniqueness Loop Pattern The template checks whether the candidate value already exists in the table. If it does, increment the suffix counter and check again. Loop until a free value is found. This pattern is extremely common in production OIM templates for generating unique account names, email addresses, etc.
Test Cases
#Test ScenarioStepsExpectedStatus
1Apply template when "BAS" already existsCreate a user with CustomProperty03 = "BAS"; create another user and apply templatesSecond user gets "BAS1"PASS
2Apply template when both "BAS" and "BAS1" existCreate another user with base value "BAS"New user gets "BAS2"PASS
3Change "BAS1" to "BAS3", re-apply on "BAS3"Update the "BAS1" user to "BAS3"; re-apply templates on that user"BAS3" updated to "BAS1" (BAS1 is now free again)PASS
4Create user that would naturally become "BAS2"Create new user with base "BAS" — BAS, BAS1 taken; BAS2 availableNew user gets "BAS4" (BAS2 was created in step 2; BAS3 now exists from step 3)PASS
Mentor Note The uniqueness suffix loop is used constantly in production for generating account names and email addresses. Make sure trainees understand the loop logic: generate candidate → check if exists → if yes, increment and repeat → if no, use it.

Requirements

Troubleshooting Reference

ProblemCauseSolution
Template not running on update Wrong trigger — "OnUpdate" not checked Check all 3 trigger checkboxes (Insert, Update, Save) in the template editor
Overwrite not working Overwrite checkbox not ticked Enable Overwrite in the template's main data settings
Template runs but value stays the same Return value equals current value — no DB write occurs Add Debug.Print statements to trace; check logic produces a different value
CCC_CustomMulti not visible in OIM Manager Schema extension not compiled and deployed Deploy Designer changes (Compile → Deploy) and restart OIM services
MultiValueProperty template not adding values Wrong column type — not configured as MultiValue in schema Verify the column is configured as VB.NET MultiValueProperty type in the schema extension
Date calculation gives wrong result Month arithmetic handled incorrectly in VB.NET Use DateAdd(DateInterval.Month, 2, EntryDate) — do not use manual arithmetic (e.g., adding 60 days)
Suffix not unique (TPL-10) Uniqueness check queries wrong table or column Ensure the WHERE clause in the uniqueness loop references the correct table and column name
