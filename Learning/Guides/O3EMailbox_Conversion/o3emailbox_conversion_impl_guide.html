O3EMailbox Shared / Regular Conversion

Implementation notes for the Exchange Online shared mailbox conversion workflow in OIM

EXO Integration Person Table Pre-configured Processes
⚠ Important — CCC_ConversionDate Not Set
CCC_ConversionDate is not populated by the conversion process because UID_Person is empty on the O3EMailbox object at the time of conversion. This is expected behaviour — do not treat the blank date as an error.

01 Config Parameters

STEP Three new parameters created under Custom\ in the OIM Config Store

All three parameters are set to Enabled and must be populated before running any conversion script or triggering a process.

Config Path Purpose Example Value
Custom\EXO\AppID Azure App Registration (Client ID) used to authenticate against Exchange Online xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
Custom\EXO\Certificate Thumbprint of the certificate installed on the OIM server, used for app-only auth ABCDEF1234567890...
Custom\EXO\Organization Exchange Online tenant domain (e.g. the .onmicrosoft.com address) contoso.onmicrosoft.com
Where to set these
In the OIM Designer: Base Data › General › Configuration Parameters. Expand Custom, locate or create the EXO sub-node, and add each parameter. Tick Enabled on each one before saving.

02 PowerShell Scripts

LOCATION 3 scripts saved in the Documents folder, each with an example_run.txt
Get
Get-O3EMailbox.ps1
Retrieves current mailbox type from Exchange Online for inspection / pre-flight checks
Convert
Convert-O3EMailbox.ps1
Converts a regular mailbox to a Shared mailbox in Exchange Online
Revert
Revert-O3EMailbox.ps1
Reverts a shared mailbox back to a Regular (user) mailbox
How to Run
  • Open the script file (.ps1) in Windows PowerShell ISE (not VS Code, not plain PowerShell console).
  • Press F5 (or click Run Script) to load the script and its functions into the session.
  • Open the matching example_run.txt in the same folder.
  • Copy the example command from the txt file and paste it into the ISE Console pane (bottom), then press Enter.
Why ISE?
These scripts use dot-sourcing patterns and rely on the ISE session to keep loaded functions in scope between calls. Running them directly (.\script.ps1) in a plain console will unload the functions immediately after execution.

03 OIM Processes

PERSON 2 pre-configured processes on the Person table
Do Not Generate These Processes
Both processes already exist and are fully configured. Do not recreate or regenerate them via the OIM Designer process generator — doing so will overwrite custom step sequences and script bindings.
CCC_O3EMailbox_toShared
Triggers on the Person table to invoke the Convert script, changing the linked O3EMailbox to a Shared mailbox in Exchange Online.
CCC_O3EMailbox_toRegular
Triggers on the Person table to invoke the Revert script, converting the linked O3EMailbox back to a Regular (user) mailbox.
Key Relationship

Both processes act on the Person table (not directly on O3EMailbox). They resolve the associated O3EMailbox via the UID_O3EMailbox foreign key on the Person record and pass the mailbox identity to the respective PowerShell script as a process step parameter.

CCC_ConversionDate Reminder
Because UID_Person is empty on the O3EMailbox object at the time these processes run, CCC_ConversionDate will remain blank — this is by design, not a defect.
