Pattern: Process Automation (not event-based) Both "Gen Conditions" are Process Automations — scheduled SQL queries that find matching objects and trigger a regular Process per object. Each stage is independent: two automations, two processes, two deduplication flags.

SQL Query Template (apply threshold and flag from table above)
UID_AADUser NOT IN (SELECT UID_AADUser FROM O3EMailbox WHERE UID_AADUser IS NOT NULL) AND XDateInserted < {THRESHOLD} AND XDateInserted > DATEADD(HOUR, -48, GETUTCDATE()) AND ISNULL({FLAG_COLUMN}, 0) = 0
O3EMailbox — Verified Table Name O3EMailbox is the Exchange Online mailbox table in OIM 9.3. It has a direct UID_AADUser FK. Do not use EX0MailBox (on-prem) or O365Mailbox.

3. COMMIT & VERIFICATION
Commit to Database
  1. Click Commit to database (Designer toolbar, top left).
  2. Confirm both automations appear in Process Orchestration → Process automation.
  3. Confirm both processes appear in Process Orchestration → Processes.
  4. Confirm custom columns exist: Schema Browser → AADUser → search CCC_Is.
End-to-End Test (Alert)
  1. Create a test AADUser with no linked O3EMailbox row.
  2. Temporarily set the threshold to 0 MINUTE in CCC_AAD_AutomationAlert.
  3. Right-click the automation → Run now.
  4. Watch Job Queue Info — both steps reach Finished.
  5. Check the Service Desk inbox. Verify tokens resolved (no literal $DisplayName$).
  6. Confirm CCC_IsFirstAlertSent = 1 in the database.
  7. Run the automation again — account must not be re-queued.
  8. Restore threshold to -1 HOUR. Repeat steps 1–7 for the escalation automation.
