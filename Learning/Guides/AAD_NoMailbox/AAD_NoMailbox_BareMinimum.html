AAD No-Mailbox Alerting
Bare Minimum Implementation
OIM 9.3  |  All improvements incorporated  |  Single notification address  |  March 2026
What this covers Everything required to run the no-mailbox alerting chain with all v1.1 improvements. Nothing more. One email address for all notifications.
Step 1 Prerequisites
Schema Extensions (AADUser table)

Add two custom Bool columns via Designer → Schema Extensions → AADUser:

CCC_IsFirstAlertSentBool (Bit), default 0
CCC_IsEscalationSentBool (Bit), default 0

After adding both: Database → Commit to database

Configuration Parameters (Base Data → Configuration parameters)
PathDefault ValuePurpose
Custom\AAD\Notifications\AlertEmailit-admin@contoso.comAll alert and escalation emails sent here
Custom\AAD\Notifications\MaxAgeHours48Ignore accounts older than this (hours). Adjust per SLA.
Mail Templates (Base Data → Mail templates → New)

Create two templates, both with base object AADUser:

NameSubject
CCC_AAD_NoMailbox_Alert[ACTION] AAD '$DisplayName$' has no EXO mailbox (created: $XDateInserted$)
CCC_AAD_NoMailbox_Escalation[ESCALATION] AAD '$DisplayName$' STILL has no EXO mailbox — 4h since $XDateInserted$

Body content: include $DisplayName$, $UserPrincipalName$, $XDateInserted$. Plain text body is acceptable.

Step 2 Generating Condition SQL (use in both automations)
Alert Automation — Generating Condition (object type: AADUser)
NOT EXISTS ( SELECT 1 FROM O3EMailbox m WHERE m.UID_AADUser = AADUser.UID_AADUser ) AND XDateInserted < DATEADD(HOUR, -1, GETUTCDATE()) AND XDateInserted > DATEADD(HOUR, -48, GETUTCDATE()) AND ISNULL(CCC_IsFirstAlertSent, 0) = 0
Escalation Automation — Generating Condition (object type: AADUser)
NOT EXISTS ( SELECT 1 FROM O3EMailbox m WHERE m.UID_AADUser = AADUser.UID_AADUser ) AND XDateInserted < DATEADD(HOUR, -4, GETUTCDATE()) AND XDateInserted > DATEADD(HOUR, -48, GETUTCDATE()) AND ISNULL(CCC_IsEscalationSent, 0) = 0
Reset Automation — Generating Condition (object type: AADUser)
EXISTS ( SELECT 1 FROM O3EMailbox m WHERE m.UID_AADUser = AADUser.UID_AADUser ) AND ( ISNULL(CCC_IsFirstAlertSent, 0) = 1 OR ISNULL(CCC_IsEscalationSent, 0) = 1 )
NOT EXISTS vs NOT IN NOT EXISTS is NULL-safe and faster. Do not switch back to NOT IN — a single NULL in O3EMailbox silently breaks the entire automation.

Step 4 Processes — Alert and Escalation (identical structure)

Build both CCC_AAD_SendNoMailboxAlert and CCC_AAD_SendNoMailboxEscalation with the same two-step structure:

Step 1 — Send_Email
FieldValue
ComponentNotifications / Mail
TemplateCCC_AAD_NoMailbox_Alert (or Escalation template for the second process)
RecipientConfig param: Custom\AAD\Notifications\AlertEmail
On errorWait — 3 retries, 10 minutes apart
Previous step resultAlways
Pre-Script (apply to both processes)
' Late-check: skip if mailbox appeared since automation queued this Dim mailboxCount As Integer = Session.Source().Count( _ "O3EMailbox", "UID_AADUser = '$UID_AADUser$'") If mailboxCount > 0 Then Return False ' Late-check: skip if account now exceeds configured age cap Dim maxHours As Integer = 48 Dim cfg As String = Session.Config().GetConfigParm("Custom\AAD\Notifications\MaxAgeHours") If IsNumeric(cfg) Then maxHours = CInt(cfg) If DateDiff("h", CDate("$XDateInserted$"), Now()) > maxHours Then Return False Return True
Step 2 — Set_Flag
FieldValue
ComponentSQL / HandleObject → UpdateData
TableAADUser
WhereClauseUID_AADUser = '$UID_AADUser$'
UpdateData (Alert process)CCC_IsFirstAlertSent=1
UpdateData (Escalation process)CCC_IsEscalationSent=1
Previous step resultOn success — flag only sets if email sent
Critical: On success dependency If Step 1 fails all retries, Step 2 must NOT run. Setting this to "On success" ensures a failed email send does not silently mark the account as alerted.
Step 5 Reset Process

Create CCC_AAD_ResetNoMailboxFlags with one step:

FieldValue
ComponentSQL / HandleObject → UpdateData
TableAADUser
WhereClauseUID_AADUser = '$UID_AADUser$'
UpdateDataCCC_IsFirstAlertSent=0,CCC_IsEscalationSent=0
Previous step resultAlways

No email. No Pre-Script. This step only resets flags when a mailbox appears, so the account can be re-monitored if the mailbox is later removed.

Done Verify
  1. Create a test AADUser with no O3EMailbox row. Run alert automation manually (right-click → Run now in Designer).
  2. Confirm process queues in Job Queue and email arrives at AlertEmail address.
  3. Confirm CCC_IsFirstAlertSent = 1 in the database.
  4. Provision O3EMailbox row. Run reset automation. Confirm both flags reset to 0.
  5. Remove SMTP config temporarily. Rerun alert automation. Confirm flag stays 0 after retries exhaust.
