1. PROCESS FLOW OVERVIEW
The Three-Stage Flow

This process chain monitors new Azure AD accounts that have not had an Exchange Online mailbox provisioned. It sends a time-based alert at 1 hour and escalates automatically at 4 hours if the issue is still unresolved.

Gen Condition
New AAD Account without EXO mailbox
for over 1 hour
Send Email
CCC_AAD_NoMailbox_Alert
Gen Cond check if needed
Gen Condition
Check if 4 hours have passed
and still no EXO mailbox
Send 2nd Email
CCC_AAD_NoMailbox_Escalation
Final automated notification
Time-Based, Not Event-Based A standard OIM Process fires once on INSERT/UPDATE and cannot re-evaluate itself after a time gap. This workflow uses the Process Automation pattern — a scheduled SQL query that finds matching objects at runtime and triggers a regular Process for each one. The same pattern is used in the SCN-3 "Email 15 Days Before Expiry" exercise.
Four-Component Architecture
#ComponentTypePurpose
1CCC_AAD_AutomationAlertProcess AutomationRuns hourly. Queries AADUser records: no O3EMailbox row, created >1hr ago, alert flag not set
2CCC_AAD_SendNoMailboxAlertRegular ProcessTriggered by Automation 1. Sends alert email. Sets CCC_IsFirstAlertSent = 1
3CCC_AAD_AutomationEscalationProcess AutomationRuns hourly. Same query but threshold >4hr, escalation flag not set
4CCC_AAD_SendNoMailboxEscalationRegular ProcessTriggered by Automation 3. Sends escalation email. Sets CCC_IsEscalationSent = 1

Mail Templates

Schema Extensions — Custom Boolean Columns on AADUser

Two custom columns are used as deduplication flags. Without them, each automation run would re-send the email. Add them via Designer Schema Extensions before building the processes.

How to Add in Designer
One Identity Manager Schema → Schema extensions → AADUser table → Add column
Column NameData TypeDefaultPurpose
CCC_IsFirstAlertSent Bool (Bit) 0 Set to 1 after the 1-hour alert is sent. Prevents duplicate sends on subsequent automation runs.
CCC_IsEscalationSent Bool (Bit) 0 Set to 1 after the 4-hour escalation is sent. Same deduplication mechanism.
Schema Compile Required After adding custom columns, run Database → Commit to database in Designer to apply the schema change to the physical database. The columns will not appear in process parameters until this step is complete.
Configuration Parameters for Recipient Addresses

Store recipient addresses as OIM Configuration Parameters. This allows routing to be changed by an operations administrator without modifying the process in Designer.

How to Add in Designer
Base Data → Configuration parameters → New

3. GEN CONDITION — 1-HOUR CHECK
Process Automation: CCC_AAD_AutomationAlert

This Process Automation runs on a schedule and queries the live database for AADUser records that have been active for more than 1 hour with no Exchange Online mailbox provisioned. It then triggers the alert process for each matching object.

Navigation
Process Orchestration → Process automation → New
FieldValue
NameCCC_AAD_AutomationAlert
Object typeAADUser
ProcessCCC_AAD_SendNoMailboxAlert (created in Section 4)
ScheduleEvery 1 hour — create under Base Data → Schedules if needed
EnabledYes
SQL Query — Generating Condition (O3EMailbox — verified schema)

The SQL query returns the UID_AADUser values of all accounts that match the 1-hour no-mailbox condition. The table O3EMailbox is the Exchange Online mailbox table and contains a direct UID_AADUser foreign key — confirmed against the OIM 9.3 database schema.

UID_AADUser NOT IN ( SELECT UID_AADUser FROM O3EMailbox WHERE UID_AADUser IS NOT NULL ) AND XDateInserted < DATEADD(HOUR, -1, GETUTCDATE()) AND XDateInserted > DATEADD(HOUR, -48, GETUTCDATE()) AND ISNULL(CCC_IsFirstAlertSent, 0) = 0
ClausePurpose
NOT IN (SELECT … FROM O3EMailbox)Excludes any AADUser that already has an O3EMailbox row — i.e., mailbox is provisioned
XDateInserted < DATEADD(HOUR, -1, …)Account must be at least 1 hour old
XDateInserted > DATEADD(HOUR, -48, …)Upper cap — ignores very old unresolved accounts (adjust to your SLA)
ISNULL(CCC_IsFirstAlertSent, 0) = 0Deduplication — only pick up accounts not yet alerted
Table Name Confirmed The correct Exchange Online mailbox table in OIM 9.3 is O3EMailbox. Do not use EX0MailBox (on-premises Exchange) or O365Mailbox. Verify in Designer → Schema Browser → O3EMailbox if uncertain.

4. SEND EMAIL — ALERT (STEP 1)
Process: CCC_AAD_SendNoMailboxAlert

This is a regular OIM Process (not an automation). It is triggered by CCC_AAD_AutomationAlert — never by a table event. It contains two process steps: send the alert email, then set the deduplication flag.

Generating ConditionLeave blank — the Automation SQL already filters correctly

SettingValue
Step nameSend_AlertEmail
ComponentNotifications / Mail
Mail template parameterCCC_AAD_NoMailbox_Alert
Recipient parameterValue of config param Custom\AAD\Notifications\ServiceDeskEmail
Pre-ScriptSee below — re-checks the mailbox state at execution time
Step 1 Pre-Script — Skip if Mailbox Appeared Since Automation Queued
' Re-verify: skip the email if a mailbox was provisioned between automation query and step execution Dim uid As String = $UID_AADUser$ Dim count As Integer = Session.Source().Count("O3EMailbox", "UID_AADUser = '" & uid & "'") If count > 0 Then Return False ' Mailbox now exists — skip step End If Return True
Step 2 — Set_AlertSentFlag
SettingValue
Step nameSet_AlertSentFlag
ComponentSQL / HandleObject
ActionSet CCC_IsFirstAlertSent = 1 on the current AADUser object
WhereClauseUID_AADUser = '$UID_AADUser$'
Why Set the Flag Even if Pre-Script Skips? Place Step 2 outside the Pre-Script scope — it should run regardless of whether the email was sent. If a mailbox appeared just in time (Step 1 skipped), setting the flag prevents the automation from re-queuing this account on the next run.

5. GEN CONDITION — 4-HOUR CHECK (STEP 2)
Process Automation: CCC_AAD_AutomationEscalation

A second Process Automation implements the 4-hour gate. It runs on the same hourly schedule as the alert automation but uses a longer time threshold and a separate deduplication flag. It triggers the escalation process only for accounts still missing a mailbox after 4 hours.

Navigation
Process Orchestration → Process automation → New
FieldValue
NameCCC_AAD_AutomationEscalation
Object typeAADUser
ProcessCCC_AAD_SendNoMailboxEscalation (created in Section 6)
ScheduleEvery 1 hour — same schedule as the alert automation
EnabledYes
SQL Query — 4-Hour Generating Condition

Identical structure to the alert query but with a 4-hour threshold and checking the escalation flag instead.

UID_AADUser NOT IN ( SELECT UID_AADUser FROM O3EMailbox WHERE UID_AADUser IS NOT NULL ) AND XDateInserted < DATEADD(HOUR, -4, GETUTCDATE()) AND XDateInserted > DATEADD(HOUR, -48, GETUTCDATE()) AND ISNULL(CCC_IsEscalationSent, 0) = 0
ClauseDifference from Alert Query
DATEADD(HOUR, -4, …)4-hour threshold instead of 1 hour
CCC_IsEscalationSentUses the escalation flag, not the alert flag — both emails can coexist independently
The Two Automations are Independent The alert and escalation automations each check their own flag. An account that received the 1-hour alert (flag = 1) will still be picked up by the 4-hour escalation automation because that automation only checks CCC_IsEscalationSent. This is by design — both emails must be sent.

7. COMMIT & VERIFICATION
Commit to Database

After completing all four components, commit the entire change set to the database in one operation.

  1. Click Commit to database on the Designer toolbar (first button, top left).
  2. Wait for the progress dialog to complete — do not close Designer during the commit.
  3. Navigate to Process Orchestration → Process automation — both CCC_AAD_AutomationAlert and CCC_AAD_AutomationEscalation should appear in the list.
  4. Navigate to Process Orchestration → Processes — both regular processes should appear.
End-to-End Test Procedure
  1. Create a test AADUser record with no linked O3EMailbox row in the non-production OIM tenant.
  2. Temporarily shorten the threshold on the alert automation from -1 HOUR to 0 MINUTE to avoid waiting:
    -- Temporary test version AND XDateInserted < DATEADD(MINUTE, 0, GETUTCDATE())
  3. Manually trigger the automation: Designer → Process automation → right-click CCC_AAD_AutomationAlert → Run now.
  4. Monitor Job Queue Info — the process should appear and both steps should reach Finished status.
  5. Check the test inbox for the alert email. Verify HTML rendering and that all $Token$ values resolved correctly.
  6. Confirm the flag: SELECT CCC_IsFirstAlertSent FROM AADUser WHERE UID_AADUser = '...' — must return 1.
  7. Run the automation again — the account must NOT be picked up a second time.
  8. Restore the threshold to -1 HOUR before go-live.
  9. Repeat steps 2–8 for the escalation automation using the -4 HOUR threshold test.
